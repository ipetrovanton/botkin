# Расширенные тезисы — E-CODE 2026, трек ML/DS

**Доклад:** Фото анализа → структура без облака: локальный VLM-пайплайн разбора медицинских
документов на одной видеокарте.
**Формат:** ~40 минут + Q&A. **Уровень:** middle+ / senior.

## Актуальность и проблема

Результаты анализов и медицинские заключения — персональные данные специальной категории
(ст. 10 152-ФЗ). Любая отправка в облачный OCR или сторонний LLM-API — это и юридический риск,
и этическая проблема. При этом задача массовая: у каждого дома копятся PDF из лабораторий и
фото бланков, и их хочется хранить структурированно и видеть динамику. Вывод, заданный до
первой строчки кода: **разбор должен быть полностью локальным**.

Это задаёт жёсткие инженерные рамки: инференс на потребительской видеокарте (а не на кластере),
никаких внешних сервисов для данных, и при этом — структурированный, пригодный к хранению и
поиску результат, а не «простыня текста».

## Метод / архитектура

Telegram-бот (aiogram 3) + backend (FastAPI) + SQLite (WAL). Распознавание — vision-модель
`qwen3-vl:8b-instruct` в Ollama, отлажено на **RTX 3080** (Ollama в WSL2). Пайплайн:
`приём → препроцессинг → классификация → извлечение → нормализация → хранение → доставка`.

Ключевые решения:
- **Vision напрямую вместо OCR.** Картинка бланка → JSON по Pydantic-схеме через `instructor`
  (`Mode.JSON`), а не «OCR → текстовая LLM». Модель «видит» таблицу как таблицу.
- **Structured output как контракт.** Ответ обязан пройти Pydantic-схему (`list[LabResult]` /
  `list[DoctorReport]`), иначе перезапрос (`max_retries`). Никакого парсинга прозы.
- **Нормализация по авторитетному справочнику.** Названия лекарств сверяются с офлайн-реестром
  ГРЛС (20 948 названий) через RapidFuzz (абсолютная дистанция Дамерау-Левенштейна + ratio-floor)
  с правилом безопасности: не прошло порог — не подменяем (защита редких препаратов).
- **Восстановимость.** Рядом с нормализованными полями хранится сырой ответ модели
  (`raw_extraction`) — данные пересобираемы при смене правил.

## Грабли (ядро доклада)

1. **`thinking` vs `instruct`.** `thinking`-вариант тратит время на цепочку рассуждений, не
   улучшая извлечение → переход на `instruct` дал кратное ускорение. [ИЗМЕРИТЬ: с/документ]
2. **Бесконечные циклы генерации** при temperature=0.0 на сложных таблицах → `repeat_penalty=1.2`.
3. **Обрыв JSON** на длинных бланках → `num_predict=8192` + списочные схемы вместо wrapper-классов
   (лечит `IncompleteOutputException`).
4. **Препроцессинг под телефон:** upscale до 2200 px, CLAHE (clip 2.0), unsharp, консервативный
   deskew (HSV-маска документа, строгие guard-условия, фолбэк на полный кадр).
5. **Windows → WSL2 loopback:** бот автоопределяет IP WSL2 (`hostname -I`), обходя
   `Connection Refused` на `localhost`/`127.0.0.1`.
6. **Атомарная доставка** результата (поллинг + push без дублей через `claim_delivery`).

## Результаты

- Рабочий локальный пайплайн: фото/PDF → структурированная карточка в Telegram, без облака.
- Офлайн-нормализация лекарств с заполнением МНН и статуса регистрации из ГРЛС.
- Инженерная дисциплина: TDD, **115 тестов** (vision мокается → CPU без GPU), ruff чист.
- Количественные замеры латентности/точности на RTX 3080 — [ИЗМЕРИТЬ] к моменту доклада
  (см. план ниже); честно покажу и сильные стороны, и ограничения.

## Новизна / польза для аудитории

Не «ещё одна обёртка над GPT», а **конкретный рецепт приватного on-prem разбора документов** на
одной видеокарте, с разбором того, где локальная VLM ломается и как это чинить. Слушатель
унесёт: критерии «vision-LLM vs OCR», набор параметров стабильного structured-output, подход к
нормализации распознанного текста по справочнику и список граблей, которые иначе придётся
собирать самому.
