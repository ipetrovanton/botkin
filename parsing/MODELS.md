# Models — реальные замеры на ThinkPad RTX 3080 16 GB

Замерено: 2026-05-28 (вечер 1)
Окружение: WSL2 Ubuntu 22.04, Ollama 0.24.0, CUDA 13.3, драйвер NVIDIA 610.47

## Qwen3-14B (chat + extract)
- **Тег:** `qwen3:14b`
- **Размер:** 9.3 GB на диске
- **VRAM:** ~9.0 GB резидентно + 1.5 GB KV cache
- **Latency (chat):** 23.8 t/s
- **Latency (extract, low-temp):** TBD на вечере 3

## Qwen3-VL-8B (рукопись, fallback)
- **Тег:** `qwen3-vl:8b`
- **Размер:** 6.1 GB
- **VRAM:** ~5.0 GB при инференсе
- **Swap-cycle (выгрузка LLM → инференс VLM → возврат LLM):** TBD (требуется тестовое изображение)

## BGE-M3 (embeddings)
- **Тег:** `bge-m3`
- **Размер:** 1.2 GB
- **Dim:** 1024
- **VRAM:** ~1.5 GB при загруженной модели
- **Throughput:** 18 chunks/min (ниже ожидаемых 1000+, требуется оптимизация)

## VRAM-бюджет в обычном состоянии (chat + extract + RAG)

| Компонент      | VRAM    |
|----------------|---------|
| Qwen3-14B      | 9.0 GB  |
| BGE-M3         | 1.5 GB  |
| KV-cache 8K    | 1.5 GB  |
| OS + Windows + дрова | 1.0 GB |
| **Активно**    | **~13 GB / 16 GB** |
| **Запас**      | ~3 GB   |

## Текущий статус VRAM
- **Занято:** 11.5 GB / 16 GB
- **Свободно:** 4.9 GB
- **Процессы Ollama:** 3 процесса активны

## Проблемы и наблюдения
1. **BGE-M3 throughput:** 18 chunks/min вместо ожидаемых 1000+. Возможные причины:
   - Особенности реализации BGE-M3 в Ollama
   - Неоптимальные параметры запроса
   - Требуется дополнительная настройка

2. **VLM тест:** Требуется тестовое изображение для полного теста hot-swap

## План Б по VRAM
Если на практике throughput Qwen3-14B окажется ниже 20 t/s или VRAM-просадки:
1. `ollama pull qwen3:7b` (~4 GB) — даёт +5 GB запас, чуть хуже на сложных вопросах
2. Уменьшить num_ctx с 8K → 4K
3. Отключить параллельный embedding во время extract
