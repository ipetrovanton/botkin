# Полное руководство по настройке GPU NVIDIA в WSL 2 с нуля

Инструкция подходит для исправления ошибки `nvidia-smi: command not found` и настройки правильной архитектуры дистрибутивов при работе параллельно с Docker Desktop.

---

## Шаг 1. Подготовка Windows и установка драйвера

Вся работа с графическим процессором в WSL 2 строится на сквозном доступе (pass-through). Драйвер устанавливается **только в хост-систему Windows**.

1. **Скачайте официальный драйвер:** Перейдите на сайт NVIDIA в раздел [Драйверы](https://nvidia.com).
2. **Тип драйвера:** Подойдет любой актуальный драйвер (Game Ready или Studio). Специальные версии для WSL больше не нужны — поддержка встроена в основные ветки.
3. **Установка:** Запустите установщик в Windows, выберите чистую установку и **перезагрузите компьютер**.

> ⚠️ **Важно:** Никогда не пытайтесь устанавливать драйверы NVIDIA (`apt-get install nvidia-driver`) внутри терминала Linux в WSL. Это гарантированно сломает графический слой виртуальной машины.

---

## Шаг 2. Установка полноценного дистрибутива Linux

Служебные дистрибутивы вроде `docker-desktop` не подходят для разработки. Нужна полноценная операционная система.

1. Откройте **PowerShell** от имени администратора.
2. Проверьте список доступных для установки систем:
   ```powershell
   wsl --list --online
   ```
3. Установите Ubuntu (самый поддерживаемый дистрибутив для работы с ИИ и GPU):
   ```powershell
   wsl --install -d Ubuntu
   ```
4. В процессе установки откроется консоль Linux. Придумайте **имя пользователя** и **пароль** (символы пароля при вводе скрыты).

---

## Шаг 3. Настройка дистрибутива по умолчанию

Чтобы команды WSL адресовались вашей рабочей операционной системе, а не служебным контейнерам Docker Desktop:

1. В **PowerShell** выполните команду назначения главного дистрибутива:
   ```powershell
   wsl --set-default Ubuntu
   ```
2. Проверьте статус командой:
   ```powershell
   wsl --list --verbose
   ```
   *Напротив `Ubuntu` должна стоять звездочка (`*`), а в столбце VERSION — цифра `2`.*

---

## Шаг 4. Проверка и привязка путей nvidia-smi в Linux

В современных версиях Windows 11 и WSL утилита `nvidia-smi` подтягивается автоматически. Если при вводе команды `nvidia-smi` внутри Ubuntu вы видите ошибку `command not found`, выполните привязку путей.

1. Откройте терминал **Ubuntu**.
2. Сообщите системе, где искать утилиту, добавив путь в конфигурационный файл вашего профиля:
   ```bash
   echo 'export PATH=\$PATH:/usr/lib/wsl/lib' >> ~/.bashrc
   source ~/.bashrc
   ```
3. Если утилита всё ещё не видна приложениям, создайте символические ссылки для системных библиотек:
   ```bash
   sudo ldconfig /usr/lib/wsl/lib
   sudo ln -s /usr/lib/wsl/lib/libcuda.so.1 /usr/lib/libcuda.so
   sudo ln -s /usr/lib/wsl/lib/libnvidia-ml.so.1 /usr/lib/libnvidia-ml.so
   ```
4. Введите команду для проверки:
   ```bash
   nvidia-smi
   ```
   *Вы должны увидеть таблицу с моделью вашей видеокарты, объёмом видеопамяти и поддерживаемой версией CUDA.*

---

## Шаг 5. Решение возможных проблем (Troubleshooting)

Если команда выдает ошибку связи с драйвером (`NVIDIA-SMI has failed because it couldn't communicate...`):

1. Закройте терминал Ubuntu.
2. Откройте **PowerShell** от имени администратора и полностью перезагрузите подсистему WSL:
   ```powershell
   wsl --shutdown
   ```
3. Снова откройте Ubuntu. Подсистема инициализирует мост с видеокартой заново.
